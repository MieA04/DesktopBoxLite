## 问题 1：REQ-SYS-001 — 移除"隐藏/恢复系统桌面图标"

### 现状

需求文档 REQ-SYS-001 要求：软件启动时自动隐藏 Windows 原生桌面图标，退出时恢复。

### 问题描述

**此功能超出 DesktopBox Lite 的业务边界**。DesktopBox Lite 是一个"图标收纳盒"——它在桌面之上叠加一层图标容器。原生桌面图标是否显示应由用户自行控制，DesktopBox 不应越俎代庖。强行隐藏/恢复桌面图标存在以下风险：
- 用户可能依赖原生桌面图标操作
- 与其他桌面管理工具（如 Wallpaper Engine、Fences）冲突
- 退出时恢复失败会导致桌面图标永久丢失

### 修改方案

**删除 REQ-SYS-001**，DesktopBox 不再干预系统桌面图标。代码中移除：
- `desktop.rs` 中 `hide_desktop_icons()` / `show_desktop_icons()`
- `lib.rs` 中 setup 时对上述函数的调用
- `lib.rs` 中对应的 Tauri 命令

### 影响范围

- 需求文档：删除 REQ-SYS-001
- 代码文件：`src-tauri/src/desktop.rs`、`src-tauri/src/lib.rs`
- 依赖：`windows-sys` 中的 Windows API 调用简化

## 问题 2：REQ-BOX-001 — 窗口定位改为"底部居中浮动面板"

### 现状

窗口设置为 `fullscreen: true`，覆盖整个主显示器。

### 问题描述

全屏透明窗口虽然视觉上是透明的，但**仍然会拦截所有鼠标事件**。这意味着：
- 桌面壁纸上的右键菜单无法弹出
- 收纳盒以外的区域完全不可点击
- 用户无法与桌面上其他元素交互

### 修改方案

**REQ-BOX-001 标题**：从"全屏透明顶层窗口"改为**"底部居中透明浮动面板"**。

| 字段 | 原值 | 新值 |
|------|------|------|
| **标题** | 全屏透明顶层窗口 | 底部居中透明浮动面板 |
| **描述** | 全屏、透明、无边框窗口 | 透明、无边框窗口，默认位于屏幕底部中央，宽度与图标网格自适应 |
| **实现方式** | fullscreen + transparent + always_on_top | `transparent = true` + `decorations = false` + `always_on_top = true`，窗口尺寸由图标网格决定，默认贴底居中 |
| **验收标准 1** | 窗口覆盖整个主显示器 | 窗口位于屏幕底部中央，宽度由图标数量和间距计算得出，不超过屏幕宽度 |
| **验收标准 2** | 窗口完全透明 | 窗口完全透明，背景不遮挡桌面壁纸 |
| **验收标准 3** | 窗口始终在最前 | 窗口始终在最前 |

新增验收标准：
- **验收标准 4**：鼠标在窗口区域外可以正常与桌面交互（右键、单击等）
- **验收标准 5**：窗口默认底部贴边，上边缘与屏幕顶部保留大量空间

### 影响范围

- `src-tauri/tauri.conf.json`：移除 `fullscreen: true`，设置默认窗口尺寸和位置
- `src/styles/default.css`：`.app-container` 改为非全屏布局
- `src/components/App.ts`：窗口初始定位逻辑
- `src/main.ts`：DOM 结构可能需要调整

## 问题 3：REQ-BOX-009 — 快捷键切换显隐状态不稳定

### 现状

按下 Ctrl+Shift+D 时，`global-shortcut` 的 `on_shortcut` 回调同时收到 `Pressed` 和 `Released` 两个事件，导致一次按键触发两次切换，窗口不断在显/隐间反复横跳。

### 修改方案

在 `on_shortcut` 回调中过滤事件类型，**仅在 `Pressed` 事件触发时执行切换**：

```rust
.on_shortcut(shortcut, move |app, _shortcut, event| {
    if event != ShortcutEvent::Pressed {
        return; // 忽略 Released 事件
    }
    // ... toggle logic
})
```

### 影响范围

- `src-tauri/src/lib.rs`：toggle shortcut 的 handler 添加事件类型判断
- 需确认 `ShortcutEvent::Pressed` 的准确拼写

## 问题 4：REQ-BOX-009 — 隐藏后窗口仍拦截鼠标事件

### 现状

执行 `window.hide()` 后，窗口视觉上消失，但它仍然是 `always_on_top` 的透明窗口，在某些情况下仍然拦截鼠标事件传递到底层桌面。

### 修改方案

根本解决方案：**窗口不设为全屏，缩小到只覆盖图标区域**（即问题 2 的修复）。当窗口不覆盖整个屏幕时，隐藏/显示只需控制窗口可见性即可，窗口之外的区域自然可以正常交互。

辅助措施：
- `window.hide()` 应确保窗口完全不可见且不接收事件
- 如果 Tauri 的 `hide()` 仍然拦截事件，考虑暂时取消 `always_on_top` 或将窗口移出屏幕外

### 影响范围

同问题 2（窗口尺寸改造）

## 问题 5：REQ-BOX-009(增强) — 显隐动画效果

### 现状

窗口显示/隐藏是瞬间切换，无过渡动画。

### 修改方案

添加 CSS 动画，要求在 **0.2s 内完成**：

**隐藏时（坠落动画）**：
1. 窗口透明度从 1 → 0
2. 窗口沿 Y 轴向下移动 60px
3. 动画完成后执行实际的 `window.hide()`

**显示时（弹出动画）**：
1. 窗口先从隐藏状态恢复为可见（底部偏移 + 透明度 0）
2. 透明度从 0 → 1
3. 窗口沿 Y 轴向上移动 60px 回到原位

### 技术实现

前端监听 `show`/`hide` 事件，使用 CSS `@keyframes` + `transition`，控制 `0.2s ease-out` 动画。

### 影响范围

- `src/styles/default.css`：新增 `@keyframes` 动画
- `src/components/App.ts`：监听 Tauri 窗口事件，触发动画
- `src-tauri/src/lib.rs`：前端通过事件监听显隐切换
