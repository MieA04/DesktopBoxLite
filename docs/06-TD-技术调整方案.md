## 问题 2：窗口改为底部居中浮动面板 (REQ-BOX-001 重定义)

### 核心变更

**窗口策略**：从全屏窗口 → 特定尺寸窗口，默认底部居中。

### `tauri.conf.json` 变更

```json
{
  "app": {
    "windows": [{
      "title": "DesktopBox Lite",
      "transparent": true,
      "decorations": false,
      "alwaysOnTop": true,
      "focusable": true,
      "skipTaskbar": true,
      "resizable": false,
      // 移除 fullscreen: true
      // 不设 width/height 默认值，让前端控制
    }]
  }
}
```

### 前端变更

**`src/main.ts`**：应用容器不再占满全屏，新增一个居中的内容区。

```html
<div class="app-container">
  <!-- 不设宽高限制，由内容撑开 -->
  <div class="resizable-window">
    <div class="search-bar"></div>
    <div class="icon-grid"></div>
  </div>
</div>
```

**`src/styles/default.css`**：
- `.app-container`：不再 `width: 100vw; height: 100vh`，改为自适应内容
- `.resizable-window`：默认底部居中定位，`top: auto; bottom: 20px; left: 50%; transform: translateX(-50%)`
- 窗口宽度由图标网格撑开，高度由搜索栏 + 网格撑开

**`src/components/App.ts`**：
- 初始窗口定位逻辑从居中改为底部居中
- 拖拽缩放后保存尺寸逻辑不变

**`src/components/ResizeHandle.ts`**：
- 初始化 `ensureExplicitPosition` 改为基于底部定位（`bottom` + `left` + `transform`）

### 数据流

```
启动 → 计算图标数量 → 计算窗口尺寸 → 窗口贴底居中
  → 用户拖拽缩放 → 保存尺寸 → 下次启动恢复
```

## 问题 3：快捷键显隐稳定修复 (REQ-BOX-009)

### 根因

`tauri-plugin-global-shortcut` 的 `on_shortcut` 回调在**按下和释放时各触发一次**（一次 `Pressed` + 一次 `Released`）。当前代码未过滤事件类型，导致一次按键触发两次切换。

### 修复方案

在 toggle handler 中加入事件类型判断：

```rust
.on_shortcut(shortcut, move |app, _shortcut, event| {
    // 仅在按下时触发，忽略释放事件
    if !matches!(event, tauri_plugin_global_shortcut::ShortcutEvent::Pressed) {
        return;
    }
    // ... toggle window visibility
})
```

### 影响范围

仅 `src-tauri/src/lib.rs` 中 toggle shortcut 的 handler。

## 问题 4：隐藏后拦截鼠标事件 (REQ-BOX-009)

### 根因

此问题由"全屏窗口"（问题 2）导致。全屏透明窗口即使 `hide()` 后，在某些情况下仍会捕获鼠标事件。**窗口缩小到图标区域后，此问题自然解决**——窗口外的桌面区域不受影响。

### 辅助措施

通过 Tauri 监听窗口 `visible` 状态变化，确保 `hide()` 后窗口确实不响应事件。如果仍有穿透问题，可添加 `window.set_always_on_top(false)` 在隐藏时暂时取消置顶。

### 影响范围

与问题 2 的窗口尺寸改造一致。

## 问题 5：显隐动画效果 (P2)

### 实现方案

前后端协作方式：

1. **后端**（`lib.rs`）：切换显隐时，不再直接调用 `window.hide()`/`window.show()`，改为向前端发送事件 `animate-hide` / `animate-show`。

2. **前端**（`App.ts`）：监听事件，触发 CSS 动画：
   - `animate-hide`：添加 `.hiding` 类 → CSS `@keyframes` 执行 0.2s 坠落动画 → 动画结束后调用 `invoke("finish_hide")` 通知后端执行 `window.hide()`
   - `animate-show`：后端先执行 `window.show()` → 前端添加 `.showing` 类 → CSS `@keyframes` 执行 0.2s 弹出动画 → 移除类

3. **CSS**（`default.css`）：
   ```css
   @keyframes slide-down {
     0% { opacity: 1; transform: translateY(0); }
     100% { opacity: 0; transform: translateY(60px); }
   }
   @keyframes slide-up {
     0% { opacity: 0; transform: translateY(60px); }
     100% { opacity: 1; transform: translateY(0); }
   }
   .resizable-window.hiding {
     animation: slide-down 0.2s ease-out forwards;
   }
   .resizable-window.showing {
     animation: slide-up 0.2s ease-out forwards;
   }
   ```

### 影响范围

- `src-tauri/src/lib.rs`：toggle handler + 新增 `finish_hide` / `finish_show` 命令
- `src/components/App.ts`：监听事件 + 触发动画
- `src/styles/default.css`：新增关键帧动画
