## 一、Issue 1：窗口可拖拽调整大小

### 方案选型

| 选项 | 描述 | 选中 |
|------|------|------|
| A. `resizable: true` + 原生缩放 | 启用 OS 原生窗口缩放 | ❌ 会显示原生缩放装饰，与自定义风格冲突 |
| B. **程序化 `setSize()` + `setPosition()`** | Tauri Window API 控制 | ✅ **选中** |
| C. CSS `resize: both` | 纯 CSS 缩放 | ❌ 仅作用于 div，不改变 OS 窗口 |

### 详细设计方案

#### 1.1 Tauri 配置变更

`src-tauri/tauri.conf.json` — 添加初始窗口尺寸：

```json
{
  "app": {
    "windows": [{
      "width": 800,
      "height": 600,
      "resizable": false
    }]
  }
}
```

保持 `resizable: false`，全程通过 SDK API 程序化控制缩放。

#### 1.2 前端窗口 API

使用 `@tauri-apps/api/window` 模块：

```typescript
import { getCurrentWindow, LogicalSize, LogicalPosition } from "@tauri-apps/api/window";
```

**关键 API**：
- `getCurrentWindow().setSize(new LogicalSize(w, h))` — 设置窗口物理尺寸
- `getCurrentWindow().setPosition(new LogicalPosition(x, y))` — 设置窗口屏幕位置
- `getCurrentWindow().outerPosition()` — 获取当前窗口位置

#### 1.3 拖拽缩放逻辑（ResizeHandle.ts 重写）

原有逻辑（CSS 方式）：
```
mousedown → 记录 startWidth/startHeight
mousemove → 计算 newWidth/newHeight → target.style.width/height = ...
mouseup → 持久化
```

新逻辑（OS 窗口方式）：
```
mousedown → 记录 startWidth/startHeight + 获取窗口位置 startWindowX/startWindowY
mousemove → 计算 newWidth/newHeight + 计算 newX/newY（W/N方向时）
         → appWindow.setPosition() + appWindow.setSize()
mouseup → 持久化（通过 onResize 回调）
```

**方向处理矩阵**：

| 方向 | width 计算 | height 计算 | 位置调整 | 说明 |
|------|-----------|------------|---------|------|
| E | `startWidth + dx` | — | 无 | 右边缘移动 |
| W | `startWidth - dx` | — | `X -= (newWidth - startWidth)` | 左边缘移动，右边缘固定 |
| S | — | `startHeight + dy` | 无 | 下边缘移动 |
| N | — | `startHeight - dy` | `Y -= (newHeight - startHeight)` | 上边缘移动，下边缘固定 |
| SE | `startWidth + dx` | `startHeight + dy` | 无 | 右下角移动 |
| SW | `startWidth - dx` | `startHeight + dy` | `X -= (newWidth - startWidth)` | 左下角移动 |
| NE | `startWidth + dx` | `startHeight - dy` | `Y -= (newHeight - startHeight)` | 右上角移动 |
| NW | `startWidth - dx` | `startHeight - dy` | `X -= (newWidth - startWidth)` `Y -= (newHeight - startHeight)` | 左上角移动 |

**最小尺寸限制**：`MIN_WIDTH = 120`，`MIN_HEIGHT = 150`（与当前一致）

#### 1.4 CSS 布局调整

`.resizable-window` 改为始终填充 Tauri OS 窗口：

```css
.resizable-window {
  width: 100%;
  height: 100%;
  /* 移除旧的 position: relative 和 overflow — 由 flexbox 处理 */
}
```

`.app-container` 保持 `100vw x 100vh`（与 WebView 尺寸一致）。

#### 1.5 初始化窗口位置（App.ts）

启动时将窗口定位到屏幕底部居中：

```typescript
const appWindow = getCurrentWindow();
const monitor = await appWindow.primaryMonitor();
if (monitor) {
  const { width: sw, height: sh } = monitor.size;
  const x = Math.round((sw - windowWidth) / 2);
  const y = Math.round(sh - windowHeight - 20); // 20px bottom padding
  await appWindow.setPosition(new LogicalPosition(x, y));
}
await appWindow.setSize(new LogicalSize(windowWidth, windowHeight));
```

## 二、Issue 2：添加专用拖拽手柄

### 方案设计

#### 2.1 DOM 结构变更

```html
<div class="resizable-window">
  <div class="drag-handle" data-tauri-drag-region></div>
  <div class="search-bar">
    <input class="search-input" placeholder="搜索图标...">
  </div>
  <div class="icon-grid"></div>
</div>
```

- `data-tauri-drag-region` **从 `.search-bar` 移除**，仅保留在 `.drag-handle` 上
- 拖拽手柄独立于搜索栏，互不干扰

#### 2.2 CSS 样式

```css
.drag-handle {
  height: 12px;
  background-color: transparent;
  cursor: grab;
  flex-shrink: 0;
  border-radius: 16px 16px 0 0;
}
.drag-handle:hover {
  background-color: rgba(255, 255, 255, 0.08);
}
.drag-handle:active {
  cursor: grabbing;
}
```

**设计要点**：
- 12px 高度，刚好够手指/鼠标抓取
- 透明背景，hover 时微亮提示
- `grab`/`grabbing` 光标提示可拖拽
- 顶部圆角与 `.resizable-window` 的 `border-radius: 16px` 对齐

## 五、风险与对策

| 风险 | 影响 | 概率 | 对策 |
|------|------|------|------|
| `setSize()` 高频调用导致闪烁 | 体验下降 | 低 | 如需优化可加 requestAnimationFrame 节流 |
